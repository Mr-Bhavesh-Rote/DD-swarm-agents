# Deep Due-Diligence Multi-Agent Research & Report Platform

Full-stack platform that runs **deep multi-agent due-diligence research** on **companies**
and **individuals** using Claude models, and produces **cited reports** with verified
`[n]` hyperlinked citations.

Every run produces two artifacts, both stored as JSON, viewable in the UI, and exportable
to PDF and Word:

- **RAW report** — complete per-agent visibility (every agent's full output + every source).
- **FINAL report** — consolidated, sectioned, citation-verified deliverable.

The pipeline is orchestrated with **LangGraph** (orchestrator–worker pattern), observed in
**Langfuse**, and the Claude model is **configurable from the UI** per run and per role/agent.

---

## Architecture

```
React + TS + Redux (Vite, MUI)            FastAPI (REST + SSE, JWT/RBAC)
  model selector · run trigger      →        enqueue job ──► RQ worker (Redis)
  swarm view · report viewer · export                            │
                                                                  ▼
                                              LangGraph StateGraph (Postgres checkpointer)
   planner → [Send fan-out] research swarm → aggregator → synthesizer ⇄ verifier → renderer
                    │ web_search · scrape_url · read_file · code_executor
                    ▼
   PostgreSQL (runs · agents · findings · sources · reports.report_json)   Langfuse (traces · eval scores · prompt registry)
```

The node graph (`backend/workflow/graph.py`):
**planner → research_agent\* (parallel) → aggregator → synthesizer ⇄ verifier → renderer.**
The verifier loops back to the synthesizer on unsupported citations, up to `MAX_REVISIONS`.

## Quick start (Docker)

```bash
cd deep-dd
cp backend/.env.example backend/.env     # fill ANTHROPIC_API_KEY, TAVILY_API_KEY, JWT_SECRET, LANGFUSE_*
docker compose up --build
# API:      http://localhost:8000  (docs at /docs)
# Frontend: http://localhost:8080
```

`migrate` runs Alembic migrations automatically before the API starts.

## Quick start (local dev)

**Backend**
```bash
cd backend
python3.11 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env                      # fill in secrets
alembic upgrade head                      # create schema
uvicorn app.main:app --reload            # API on :8000
python worker.py                          # background worker (separate shell)
```

**Frontend**
```bash
cd frontend
npm install
cp .env.example .env                      # VITE_API_BASE_URL=http://localhost:8000
npm run dev                               # Vite on :5173
```

## Headless run (Milestone 1 — no API/DB/UI needed)

## Configuration (`.env`, §3.1)

All secrets/settings are read from `backend/.env` via typed `pydantic-settings`
(`app/core/config.py`). The app **fails fast** on missing required keys
(`ANTHROPIC_API_KEY`, `DATABASE_URL`, `JWT_SECRET`). No secret reaches the client; the
frontend only sees `VITE_`-prefixed values. See `.env.example` for the full list.

## Workflow config (YAML, §5.7)

Agent role/goal/tools and the task DAG are externalized under `config/` as CrewAI-style
pairs (`agents.<type>.yaml` + `tasks.<type>.yaml`). One loader
(`workflow/config_loader.py`) normalizes them into the runtime `WorkflowPlan`, validating
unknown agent references, dependency cycles, and unknown tool names. Legacy `.json` plan
exports import on the same path. Plans can also be generated by the orchestrator or edited
from the UI (`GET/PUT /api/runs/{id}/plan`).

## Model selection (§4.3)

Resolution precedence: **per-agent override → per-role default → run-level global default →
system default (`claude-opus-4-8`)**. The catalog is server-driven (`GET /api/models`) so
models can be added without a UI deploy.

## Citations & verification (§4.2, §14)

- Global citation registry dedupes sources by canonical URL and assigns stable `[n]` ids.
- Every FINAL claim ends with `[n]` markers that render as hyperlinks to the source URL.
- The verifier runs an LLM-as-judge faithfulness check per cited claim against the stored
  source text and computes **citation coverage** + **faithfulness** scores (pushed to
  Langfuse as eval scores). Unsupported claims trigger a revise loop or are dropped/labelled
  `[unverified]`/`[estimate]`.

## Compliance (§10, §14)

Public sources only; every claim cited; estimates labelled. Reports carry a standing
disclaimer that they are decision-support and must be human-reviewed. The plan-approval
gate (`PUT .../plan`) and "mark reviewed" action (`POST .../review`) provide human gates.

## Acceptance criteria → where it lives (§11)

| # | Requirement | Location |
|---|---|---|
| 1 | React+TS+Redux UI, live progress + reports | `frontend/src` |
| 2 | FastAPI + LangGraph + SSE + Langfuse | `backend/app`, `backend/workflow`, `app/core/observability.py` |
| 3 | Postgres persistence | `app/db`, `app/db/alembic` |
| 4 | RAW+FINAL as JSON (`reports.report_json`) | `workflow/runner.py`, `app/db/models.py` |
| 5 | Viewable + PDF/Word export with `[n]` links | `components/ReportViewer`, `exporters/` |
| 6 | UI model selection (run + role/agent) | `components/ModelSelector`, `workflow/models.py` |
| 7 | Verified citations | `workflow/nodes/verifier.py`, `workflow/citations.py` |
| 8 | RAW + FINAL formats, both subject types | `workflow/nodes/{renderer,synthesizer}.py` |
| 9 | `.env` typed settings, fail-fast, git-ignored | `app/core/config.py`, `.gitignore` |
| 10 | YAML config + single prompt template | `config/`, `workflow/config_loader.py`, `app/core/prompts.py` |
```
